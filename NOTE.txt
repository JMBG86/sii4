Cria uma aplicação web completa, preparada para deployment na Vercel, com dashboard, autenticação segura, base de dados, API, e interface para gestão de inquéritos crime em investigação.

A aplicação deve ser desenvolvida com as seguintes características técnicas:

1. Stack tecnológica

Next.js 14 / App Router

Base de dados: PostgreSQL (usando Prisma ORM)

API: Route Handlers do Next.js

UI: Tailwind CSS + shadcn/ui

Autenticação: NextAuth com login seguro (Email/Password ou OAuth interno)

Deployment: App deve ser compatível com Vercel sem configurações adicionais

Gráficos no dashboard: Recharts ou Chart.js

2. Objectivo da aplicação

Construir um sistema interno para uma equipa de investigação criminal, destinado a controlar o fluxo dos inquéritos crime, o seu estado, diligências, remessas ao tribunal e pedidos externos a entidades diversas.
A aplicação deve permitir visualizar, gerir e exportar os inquéritos.

3. Funcionalidades principais
3.1 Dashboard

Mostrar contadores dos inquéritos por estado:

“Por iniciar”

“Em diligências”

“Remetido a Tribunal”

“Aguarda resposta”

“Inquéritos concluídos”

“De relevo”

Gráficos visuais (de barras, circulares, etc.) sobre:

Estado dos inquéritos

Inquéritos no tribunal vs na equipa

Diligências pendentes por entidade

Dashboard acessível apenas após login.

3.2 Menu lateral

Com as seguintes páginas:

Dashboard

Inquéritos

Adicionar Inquérito

Diligências

Relações entre Inquéritos

Exportações

Administração / Configurações

3.3 Página: “Inquéritos”

Tabela com:

NUIPC

Tipo de crime

Datas (ocorrência, participação, início da investigação)

Estado

Classificação (normal / de relevo)

Entidades com pedidos pendentes

Localização física (equipa / tribunal)

Cada linha deve ter cor automática consoante o estado:

Azul → Por iniciar

Amarelo → Em diligências

Vermelho → Urgente ou relevo

Verde → Concluído

Roxo → Remetido a tribunal

Opções por inquérito:

Editar

Alterar estado

Adicionar diligências

Associar/ligar a outro inquérito

Ver histórico do processo

3.4 Página: “Adicionar Inquérito”

Formulário com os campos base:

NUIPC (texto único)

Data da ocorrência

Data da participação

Data do início da investigação

Tipo de crime

Estado inicial

Classificação (normal / de relevo)

Observações

O sistema deve permitir adicionar novos campos no futuro sem quebrar a estrutura (modelo de dados extensível).

3.5 Estados do inquérito

Os inquéritos devem poder assumir estados como:

Por iniciar

Em diligências

Diligências específicas (listadas no inquérito)

Remetido a tribunal

Indicar se é para solicitar diligências X

Ou com relatório Y

Remetido a entidade externa

A aguardar resposta

Concluído (inquéritos enviados ao tribunal sem retorno previsto)

Cada alteração de estado deve gerar histórico.

3.6 Relação entre inquéritos

O sistema deve permitir:

Associar dois ou mais inquéritos entre si

Registrar a razão da ligação (ex.: mesmo suspeito)

Ver mapa de relações (lista simples ou grafo simples)

3.7 Gestão de diligências

Cada diligência deve conter:

Descrição

Entidade responsável

Data da solicitação

Data prevista de resposta

Estado (pendente / respondida / recusada)

Documento associado (opcional)

3.8 Exportações

Criar área onde o utilizador pode exportar:

Todos os inquéritos

Inquéritos com diligências pendentes

Inquéritos concluídos

Inquéritos de relevo

Formato:

CSV

Excel (opcional)

PDF (gerado server-side)
